# Outputs Reference

All examples generate output in:

```text
outputs/<example_id>/
```

Reference snapshots are versioned in:

```text
samples/expected_outputs/<example_id>/
```

## Standard files

- `run_metadata.json`: execution metadata and timing context.
- `input_text.txt`: raw task/input text used for the run.
- `example_output.txt`: final output text generated by the example.
- `transcript.md`: markdown transcript with all message events.
- `result.json`: full structured payload of all messages and final message.
- `tool_calls.jsonl`: line-delimited tool call request/execution events.

## File Purpose Matrix

| File | What it stores | Why it exists |
|---|---|---|
| `run_metadata.json` | Identifiers, timestamp, counters, stop reason | Run traceability and reproducibility |
| `input_text.txt` | The exact example input text | Makes scenario intent explicit |
| `example_output.txt` | Final textual output of the run | Fast expected-output check |
| `transcript.md` | Human-readable chronological transcript | Easy debugging and teaching |
| `result.json` | Machine-readable complete message structure | Automated validation and integrations |
| `tool_calls.jsonl` | One tool call event per line | Tool auditing and troubleshooting |
